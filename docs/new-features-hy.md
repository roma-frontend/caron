# Caron — Նոր ֆունկցիաներ (ուղեցույց)

Այս փաստաթուղթը նկարագրում է վերջին թարմացումներով ավելացված բոլոր ֆունկցիաները՝ ինչպես հաճախորդի, այնպես էլ ադմինի համար։

---

## 1. Բոնուսային ծրագիր (լոյալություն)

### Կարգավորում (ադմին)
`/admin/settings` → ներդիր **«Առաքում»** → քարտ **«Բոնուսային ծրագիր»**:
- **Միացնել բոնուսային բալերը** — ընդհանուր անջատիչ։ Անջատած վիճակում ոչ մի բալ չի հաշվարկվում/ցուցադրվում։
- **Cashback պատվերից (%)** — քանի տոկոս է վերադառնում առաքված պատվերից։
- **Բալ կարծիքի համար** — քանի բալ է ստանում հաճախորդը հաստատված կարծիքի համար։
- **Բոնուս լուսանկարով կարծիքի** — լրացուցիչ բալ, եթե կարծիքին կցված է լուսանկար։
- Պահպանել՝ **«Պահպանել կարգավորումները»** կոճակով։

### Ինչպես է աշխատում
- **Հաշվարկ**: երբ պատվերը ստանում է **«Առաքվել է»** կարգավիճակ (`/admin/orders`), հաճախորդին հաշվարկվում է cashback-ը։ Պատվերի չեղարկումը հետ է վերցնում հաշվարկված բալերը։
- **Կարծիքի բալ**: `/admin/reviews`-ում կարծիքը հաստատելիս հեղինակին ավտոմատ հաշվարկվում են բալերը (բազային + բոնուս լուսանկարի համար)։
- **Ծախսում**: վճարման էջում (checkout) մուտք գործած հաճախորդը տեսնում է **«Բալերով վճարում»** բլոկը՝ կարող է մուտքագրել բալերի քանակը (1 բալ = 1 ֏) և նվազեցնել գումարը։
- **Մնացորդ**: անձնական էջում `/dashboard` → քարտ **«Բոնուսային բալեր»** (ընթացիկ մնացորդ + ընդհանուր վաստակած)։

---

## 2. Հրավիրիր ընկերոջ (ռեֆերալ ծրագիր)

- Անձնական էջում `/dashboard` → քարտ **«Հրավիրիր ընկերոջ»**՝ ձեր անձնական կոդը և **«Պատճենել հղումը»** կոճակը (հղում՝ `…/register?ref=ԿՈԴ`)։
- Ընկերը գրանցվում է հղումով — կոդը ինքնաշխատ լրացվում է, ձևի վրա երևում է «🎁 Հրավերի կոդ»։
- Ընկերոջ **առաջին առաքված պատվերից** հետո երկուսդ էլ ստանում եք **100 բալ** (բոնուսային ծրագիրը պետք է միացված լինի)։

---

## 3. Կարծիքներ՝ լուսանկարներով

### Հաճախորդ (ապրանքի էջում)
- **«Ապրանքի գնահատում»** բաժին → **«Ավելացնել»**։
- Ընտրում եք աստղեր, գրում տեքստ, կցում **մինչև 5 լուսանկար**։
- **«Ստուգված գնում»** պիտակը ավտոմատ դրվում է, եթե մուտք գործած հեղինակը ունի այս ապրանքով պատվեր։
- **Վարկանիշի գծապատկեր** (5★…1★) ցույց է տալիս բաշխվածությունը։
- Ֆիլտր՝ **«Լուսանկարով»**, դասավորում՝ **«Օգտակար / Նոր»**, **«Օգտակար»** կոճակ (քվեարկություն)։

### Ադմին
- `/admin/reviews` → հաստատել / ջնջել։ Հաստատելը նաև հաշվարկում է բոնուսային բալերը։

---

## 4. Հարցեր ապրանքի մասին (Q&A)

### Հաճախորդ
- Ապրանքի էջում **«Հարցեր ապրանքի մասին»** → **«Տալ հարց»** → գրում է անունը և հարցը։

### Ադմին
- `/admin/qa` → գրել պատասխանը և **«Պատասխանել»** (հրապարակում է հարց+պատասխան), կամ հաստատել/ջնջել։

---

## 5. Անհատական առաջարկներ և հավաքածուներ (ապրանքի էջում)

- **«Հաճախ գնում են միասին»** — ընթացիկ ապրանք + սպասարկող ապրանքներ, checkbox-երով, ընդհանուր գին և **«Ավելացնել ընտրվածը»**։
- **«Ձեզ համար»** — անհատական ընտրանի՝ ըստ ձեր դիտած ապրանքների կատեգորիաների։

---

## 6. Կրկնել պատվերը (Buy again)

- `/orders` և `/dashboard` → **«Կրկնել պատվերը»** կոճակ. ավելացնում է պատվերի ապրանքները զամբյուղ՝ առկայության ստուգմամբ։

---

## 7. Վերադարձ / Փոխանակում

### Հաճախորդ
- `/orders` → առաքված պատվերի մոտ **«Վերադարձ»** կոճակ → բացվում է պատուհան՝
  - ընտրում եք **Վերադարձ** կամ **Փոխանակում**,
  - նշում եք ապրանքները,
  - ընտրում պատճառը և գրում մեկնաբանություն,
  - **«Ուղարկել հայտը»**։
- Հայտի կարգավիճակը երևում է նույն պատվերի մոտ (Քննարկվում է / Հաստատված / Մերժված / Ավարտված)։

### Ադմին
- `/admin/returns` → բոլոր հայտերը. **Հաստատել / Ավարտել / Մերժել**։

---

## 8. Push ծանուցումներ (բրաուզերային)

### Հաճախորդ
- `/dashboard` → **«Միացնել push ծանուցումները»**. բրաուզերը հարցնում է թույլտվություն։
- Միացնելուց հետո պատվերի կարգավիճակի փոփոխության դեպքում հաճախորդը ստանում է push ծանուցում։

### Կարգավորում (պարտադիր է աշխատելու համար)
Web-push-ը պահանջում է VAPID բանալիներ։ Առանց դրանց ֆունկցիան անջատված է (no-op)։

1. Գեներացնել բանալիները՝
   ```bash
   npx web-push generate-vapid-keys --json
   ```
2. Convex deployment-ում սահմանել՝
   ```bash
   npx convex env set VAPID_PUBLIC_KEY <public-key>
   npx convex env set VAPID_PRIVATE_KEY <private-key>
   npx convex env set VAPID_SUBJECT mailto:admin@caron.am
   ```
3. `.env.local`-ում (բրաուզերի համար)՝
   ```
   NEXT_PUBLIC_VAPID_PUBLIC_KEY=<նույն public-key>
   ```
4. Վերագործարկել՝ `npm run dev` / վերա-deploy։

> Ֆայլեր՝ `public/sw.js` (service worker), `convex/push.ts` (բաժանորդագրություն), `convex/pushNode.ts` (ուղարկում)։

## 9. Ավտոմեքենայի ընտրություն / «Իմ ավտոտնակ»

- Ավտոյի ընտրիչում՝ Մակնիշ/Մոդել/Տարի → **«Գտնել»** (փնտրում և պահում է), կամ **«Ավելացնել իմ ավտոտնակ»** (պահում առանց անցնելու)։
- Պահված մեքենաները ցուցադրվում են չիպերով. սեղմելը՝ դարձնում ակտիվ, ✕-ը՝ հեռացնում։
- Քարտերի «համապատասխանում է» պիտակը հաշվարկվում է ակտիվ մեքենայով։

---

## 10. Որոնում ձայնով

- Որոնման պատուհանում (խոշորացույց / ⌘K) աջ կողմում **խոսափողի** կոճակ. սեղմեք և խոսեք (լեզու՝ hy-AM)։
- Աշխատում է Web Speech API-ով աջակցող բրաուզերներում (օր.՝ Chrome)։

---

## 11. Ակցիաների հետհաշվարկ (countdown)

- `/promotions` → յուրաքանչյուր ակցիայի քարտի վրա կենդանի հետհաշվարկ՝ մինչև ակցիայի ավարտը, թարմացվում է ամեն վայրկյան։

---

## 12. Առաքման ժամկետ / ամսաթիվ

### Կարգավորում (ադմին)
`/admin/settings` → ներդիր **«Առաքում»**:
- **«Ժամկետ՝ Երևան/Մարզեր»** — տեքստային ժամկետ (օր.՝ `1-2 օր`)։
- **«Առաքման օրեր (թվով)»** — եթե նշված է թիվ (օր.՝ `2`), քարտի վրա ցուցադրվում է **իրական ամսաթիվ** («Առաքում մինչև 21 Հունիս»)։ `0` = ցույց տալ տեքստային ժամկետը։

### Քարտի վրա
- Ապրանքի քարտին «Առաքում …» տողը ցույց է տալիս կամ ամսաթիվը, կամ տեքստային ժամկետը։

---

## 13. Զամբյուղ և քարտ (WB-ոճ)

- Քարտին՝ խոշոր գին, լցված զեղչի պիտակ, հին գինը՝ վրան գծած, կոմպակտ վարկանիշ, առաքման տող, ինլայն զամբյուղի +/− կառավարիչ։
- Զամբյուղում՝ ապրանքների ընտրություն (checkbox), «Ընտրել բոլորը», խմբային ջնջում՝ undo-ով, անվճար առաքման progress bar, բալերի ծախսման բլոկ։

---

### Տեխնիկական նշումներ
- Բոլոր նոր Convex ֆունկցիաները/սխեման deploy-ի համար՝ `npx convex deploy` (dev-ում՝ `npx convex codegen`/`npx convex dev`)։
- Կարծիքների լուսանկարների վերբեռնումը՝ առանձին, rate-limit-ով `/api/review-upload` (հանրային)։ Admin նկարների վերբեռնումը՝ `/api/upload` (միայն admin)։
